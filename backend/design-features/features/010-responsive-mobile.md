# 010 - Адаптивный дизайн и мобильная версия

**Приоритет:** MVP (010)
**Категория:** UI/UX и доступность
**Зависимости:** Все основные фичи (001-009)

## Описание

Mobile-first адаптивный дизайн для всех страниц платформы с оптимизацией под различные устройства и размеры экранов.

## Функциональные требования

### 1. Breakpoints

```css
/* Mobile первый подход */
xs: 0-575px      // Мобильный (портретный)
sm: 576-767px    // Мобильный (ландшафт)
md: 768-1023px   // Планшет
lg: 1024-1279px  // Десктоп малый
xl: 1280-1919px  // Десктоп стандартный
xxl: 1920px+     // Десктоп широкий
```

### 4. Адаптивные компоненты

#### 4.1 Сетка видео

**Mobile (xs-sm):**
- 1 колонка, вертикальный список
- Полноширинные карточки

**Tablet (md):**
- 2-3 колонки
- Компактные карточки

**Desktop (lg+):**
- 4-6 колонок в зависимости от ширины
- Hover эффекты

#### 4.2 Видеоплеер

**Mobile:**
- Автоповорот в landscape при fullscreen
- Gesture controls (свайпы)
- Увеличенные кнопки (48x48px минимум)
- Упрощенная панель управления

**Tablet/Desktop:**
- Полная панель управления
- Клавиатурные шортка ты
- Hover элементы

#### 4.3 Формы

**Mobile:**
- Стековая раскладка (1 колонка)
- Крупные поля ввода
- Встроенная валидация
- Нативные клавиатуры (email, number, tel)

**Desktop:**
- 2 колонки где уместно
- Inline валидация
- Клавиатурная навигация

### 5. Touch оптимизация

**Минимальные размеры touch targets:**
- Кнопки: 48x48px
- Чекбоксы: 40x40px
- Ссылки: минимум 32px высота

**Жесты:**
- **Swipe left/right** - навигация между страницами
- **Swipe down** - обновление ленты (pull-to-refresh)
- **Swipe up** - мини-плеер
- **Pinch to zoom** - для изображений
- **Double tap** - like/pause видео
- **Long press** - контекстное меню

### 6. Производительность на мобильных

**Оптимизации:**
- Lazy loading изображений
- Reduced quality images на медленном соединении
- Prefetch критичных ресурсов
- Code splitting по роутам
- Service Worker для offline
- WebP формат изображений

**Индикаторы соединения:**
- Определение скорости (navigator.connection)
- Предупреждение о медленном интернете
- Режим экономии трафика
- Отключение автопревью на медленном соединении

- **Screen reader** поддержка
- **Dynamic type** - масштабирование текста

- **Reduce motion** - отключение анимаций
- **Voice control** - голосовое управление

## Технические детали

### CSS подход
```css
/* Mobile первый */
.component {
  /* Мобильные стили по умолчанию */
  flex-direction: column;
}

/* Планшеты и выше */
@media (min-width: 768px) {
  .component {
    flex-direction: row;
  }
}

/* Десктоп */
@media (min-width: 1024px) {
  .component {
    /* Десктоп стили */
  }
}
```

### Обнаружение устройства
```typescript
// Утилита
const isMobile = /iPhone|iPad|iPod|Android/i.test(navigator.userAgent);
const isTablet = /iPad|Android/i.test(navigator.userAgent) && !window.matchMedia("(max-width: 767px)").matches;
const isDesktop = !isMobile && !isTablet;

// Или медиа-запросы
const isMobileView = window.matchMedia("(max-width: 767px)").matches;
```

### Viewport meta tag
```html
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">
```

## UI/UX требования

### Приоритеты для мобильных
1. **Скорость** - быстрая загрузка
2. **Простота** - минимум кликов до контента
3. **Читаемость** - крупный текст (16px минимум)
4. **Touch-friendly** - крупные кнопки
5. **Offline** - базовая работа без интернета

### Анимации
- Используй `transform` и `opacity` (GPU)
- 60 FPS на анимациях
- Reduce motion для accessibility
- Короткие анимации (200-300ms)

## Метрики успеха
- Mobile traffic > 60%
- Mobile LCP < 2.5s
- Mobile FID < 100ms
- Mobile CLS < 0.1
- PWA install rate > 10%
- Mobile engagement time > 80% от desktop
