# 001 - Система аутентификации и авторизации

**Приоритет:** MVP (001)
**Категория:** Безопасность и пользователи
**Зависимости:** Нет

## Описание

Полноценная система регистрации, входа и управления сессиями пользователей. Базовая фича для всей платформы.

## Функциональные требования

### 1. Регистрация пользователей

#### 1.1 Email регистрация
- Поля: Email, Пароль, Подтверждение пароля, Имя пользователя
- Валидация email в реальном времени
- Требования к паролю: минимум 8 символов, буквы и цифры
- Проверка уникальности email и username
- Отправка письма с подтверждением email
- Автоматический вход после подтверждения

#### 1.2 OAuth регистрация (Google, VK, Yandex)
- Кнопки "Продолжить с Google", "Войти через VK", "Войти через Яндекс"
- Автоматическое создание профиля из данных OAuth
- Связывание существующего аккаунта по email
- Возможность добавить OAuth к существующему аккаунту

#### 1.3 Валидация на клиенте
- Проверка формата email
- Индикатор силы пароля
- Совпадение паролей
- Проверка длины username (3-30 символов)
- Запрещенные символы в username
- Real-time проверка занятости username

### 2. Вход в систему

#### 2.1 Email вход
- Вход по email или username
- "Запомнить меня" (refresh token на 30 дней)
- Ссылка "Забыли пароль?"
- Защита от bruteforce (rate limiting)

#### 2.2 OAuth вход
- Те же провайдеры что и при регистрации
- Автоматическое связывание аккаунтов

#### 2.3 Двухфакторная аутентификация (2FA)
- Опциональная настройка 2FA через TOTP (Google Authenticator)
- Резервные коды восстановления
- SMS код (опционально, в будущем)

### 3. Восстановление пароля

- Форма ввода email
- Отправка ссылки сброса пароля на email
- Страница установки нового пароля
- Срок действия ссылки: 1 час
- Уведомление на email о смене пароля

### 4. Управление сессиями

#### 4.1 JWT токены
- Access token (15 минут)
- Refresh token (30 дней или до выхода)
- HTTP-only cookies для refresh token
- Автоматическое обновление access token

#### 4.2 Активные сессии
- Список всех активных устройств/браузеров
- Информация: устройство, браузер, IP, время последней активности
- Кнопка "Выйти на всех устройствах"
- Возможность завершить конкретную сессию

### 5. Выход из системы

- Кнопка "Выйти" в меню пользователя
- Очистка токенов
- Редирект на главную страницу
- Опция "Выйти на всех устройствах"

## Адаптивность

### Мобильная версия
- Полноэкранные формы
- Увеличенные кнопки (минимум 48px высота)
- Автофокус на первом поле
- Клавиатура по типу поля (email, текст, цифры)
- Кнопки OAuth в столбик на узких экранах

### Планшет
- Центрированная форма (макс. 500px)
- Кнопки OAuth в ряд

### Десктоп
- Центрированная форма (макс. 450px)
- Все элементы в одной колонке

## Безопасность

### Клиентская часть
- XSS защита (sanitize всех полей)
- CSRF токены
- Secure, HTTP-only cookies
- SameSite=Strict для cookies
- Очистка форм после успеха

### Серверная часть
- Bcrypt хеширование паролей (cost 12)
- Rate limiting на эндпоинты авторизации
- Email verification обязателен
- Защита от timing attacks при проверке
- Логирование всех попыток входа

## Метрики успеха

- Время регистрации < 2 минут
- Конверсия регистрации > 60%
- Доля OAuth регистраций > 40%
- Успешных входов > 95%
- Время входа < 3 секунд
- Доля включенной 2FA > 10% (цель)

## Особенности для русскоязычной аудитории

- Поддержка кириллицы в username
- Провайдеры VK и Yandex наравне с Google
- Русские тексты ошибок и уведомлений
- Формат телефона +7 (для будущей SMS 2FA)
- Ссылка на политику в соответствии с законодательством РФ
