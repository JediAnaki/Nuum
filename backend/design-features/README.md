# Дизайн фич видеоплатформы

Эта директория содержит описание и планы реализации всех фич платформы.

## Структура

### `/features`
Описания фич - что они из себя представляют, для чего нужны, какую проблему решают.

Каждая фича описывается в отдельном файле с именем `[номер]-[название].md`

Пример структуры описания:
```markdown
# Название фичи

## Описание
Что это такое и зачем нужно

## Проблема
Какую проблему решает

## Целевая аудитория
Кто будет использовать

## Приоритет
Высокий / Средний / Низкий

## Зависимости
От каких других фич зависит
```

### `/implementation`
Конкретные планы реализации - что нужно создать, как, какие компоненты, API endpoints, и т.д.

Каждый план реализации соответствует фиче и имеет имя `[номер]-[название]-implementation.md`

Пример структуры плана:
```markdown
# План реализации: Название фичи

## Frontend компоненты
- Список компонентов
- Их назначение

## Backend API
- Endpoints
- Request/Response форматы

## База данных
- Таблицы
- Изменения схемы

## Зависимости
- Внешние библиотеки
- Сервисы

## Этапы реализации
1. Шаг 1
2. Шаг 2
...

## Оценка времени
X часов/дней
```

## Нумерация

Фичи нумеруются по приоритету:
- 001-099: Критические (MVP)
- 100-199: Высокий приоритет
- 200-299: Средний приоритет
- 300+: Низкий приоритет / Будущие улучшения
